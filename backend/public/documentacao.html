Início

API REST do sistema Controle Financeiro Inteligente. Todas as respostas são em JSON. Endpoints protegidos exigem o header Authorization com o token JWT (ou apenas o token).

URL base e formato

Base URL (desenvolvimento):

http://localhost:3000

Todas as requisições devem usar Content-Type: application/json quando houver body. Respostas de erro seguem o formato: 

{
  "message": "Descrição do erro"
}

Autenticação (JWT)

Endpoints marcados com requer token precisam do usuário autenticado. Primeiro faça login para obter o token.

POST /auth/login

Retorna o token JWT e os dados do usuário (sem senha).

Body (JSON):

{
  "email": "usuario@exemplo.com",
  "password": "sua_senha"
}

Resposta 200:

{
  "token": "eyJhbGciOiJIUzI1NiIs...",
  "user": {
    "id": "uuid",
    "name": "Nome",
    "email": "usuario@exemplo.com",
    "createdAt": "...",
    "updatedAt": "..."
  }
}

Uso do token:

Authorization: Bearer eyJhbGciOiJIUzI1NiIs...

Ou apenas o token no header:

Authorization: eyJhbGciOiJIUzI1NiIs...
Código Descrição
200Login realizado com sucesso
400E-mail e senha são obrigatórios
401Credenciais inválidas

Swagger UI

Para testar a API de forma interativa, use o Swagger em /docs.

Usuários

Cadastro e gestão de usuários. Apenas POST /users é público.

POST /users

Criar usuário (público). Body: name, email, password.

{
  "name": "Fulano",
  "email": "fulano@exemplo.com",
  "password": "123456"
}
GET /users requer token

Listar todos os usuários.

GET /users/{id} requer token

Buscar usuário por ID.

PUT /users/{id} requer token

Atualizar usuário (name, email, password — parciais).

DELETE /users/{id} requer token

Remover usuário.

Transações recorrentes

Todas as rotas exigem autenticação. O userId é definido pelo token.

POST /recurring-transactions requer token

Criar transação recorrente.

{
  "title": "Assinatura Streaming",
  "amount": 49.9,
  "type": "EXPENSE",
  "frequency": "MONTHLY",
  "startDate": "2026-03-01T00:00:00.000Z",
  "endDate": null,
  "active": true
}

type: INCOME | EXPENSE | TRANSFER

frequency: DAILY | WEEKLY | MONTHLY | YEARLY

GET /recurring-transactions requer token

Listar transações recorrentes do usuário.

GET /recurring-transactions/{id} requer token

Buscar por ID.

PUT /recurring-transactions/{id} requer token

Atualizar (campos parciais).

DELETE /recurring-transactions/{id} requer token

Remover transação recorrente.

Parcelas (Installment)

Parcelas vinculadas a transações. Todas as rotas exigem token. Só são listadas/alteradas parcelas de transações do usuário.

POST /installments requer token

Criar parcela.

{
  "number": 1,
  "total": 12,
  "amount": 99.9,
  "dueDate": "2026-03-15T00:00:00.000Z",
  "paid": false,
  "transactionId": "uuid-da-transacao"
}
GET /installments requer token

Listar parcelas das transações do usuário.

GET /installments/{id} requer token

Buscar parcela por ID.

PUT /installments/{id} requer token

Atualizar (number, total, amount, dueDate, paid — parciais).

DELETE /installments/{id} requer token

Remover parcela.

Experimentos A/B

Criação de experimentos com variantes (ex.: controle vs. variante A) e atribuição de usuários a uma variante. Todas as rotas exigem token.

POST /experiments requer token

Criar experimento com variantes. Body: name, slug, description?, startDate, endDate?, active?, variants: [{ name, slug }].

GET /experiments requer token

Listar experimentos.

GET /experiments/{id} requer token

Buscar experimento por ID.

POST /experiments/{id}/assign requer token

Obter ou atribuir a variante do experimento para o usuário logado (aleatória na primeira vez).

GET /experiments/{id}/assignment requer token

Consultar variante já atribuída ao usuário.

Métricas (CTR, tempo no app)

Registro de eventos e consulta de métricas por experimento: CTR (taxa de cliques = cliques/impressões) e tempo no app (sessões).

POST /metrics/events requer token

Registrar evento. Body: eventType (CLICK | IMPRESSION | SCREEN_VIEW | SESSION_START | SESSION_END), target?, experimentId?, variantId?, metadata?.

{
  "eventType": "CLICK",
  "target": "primary_button",
  "experimentId": "uuid",
  "variantId": "uuid"
}
{
  "eventType": "SESSION_END",
  "experimentId": "uuid",
  "variantId": "uuid",
  "metadata": { "durationSeconds": 120 }
}
GET /metrics/experiments/{id}/ctr requer token

CTR por variante (impressões, cliques, ctr).

GET /metrics/experiments/{id}/time-in-app requer token

Tempo no app por variante (totalSeconds, sessionCount, averageSecondsPerSession).

GET /metrics/experiments/{id}/summary requer token

Resumo agregado: CTR + tempo no app por variante.
